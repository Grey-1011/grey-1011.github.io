MoneyBox项目总结-Vue@cli引入svg

Created2022-03-08|Updated2022-03-10
|Post View:

引入svg的简单方法

  • 在iconfont上选好图标加入购物车，添加至项目，然后拷贝项目下面生成的symbol代码：
  • 然后引用，就可以使用svg标签
  • 上述方法虽然简单，但是每次更新svg图标都需要登录iconfont，获取新的svg.js,重新引入项目，才可
  • 解决方法：通过webpack引入svg,直接下载svg图标，复制到项目

如何使用 vue@cli引入svg

  • 在iconfont 下载图标，复制到项目文件 src/assets/icons

    安装 svg loader

  • yarn add svg-sprite-loader -D

配置vue.config.js

  • 官方文档中的配置是写在webpack.config.js，而我们使用的项目是 vue@cli搭建的，只有Vue.config.js,所以需要把官方文档里的配置翻译成 vue.config.js 可使用的
  • vue官方文档给出，可以在vue.config.js里用一个chainWebpack来修改webpack的配置，且官方给出的栗子是：https://cli.vuejs.org/zh/guide/webpack.html#%E9%93%BE%E5%BC%8F%E6%93%8D%E4%BD%9C-%E9%AB%98%E7%BA%A7
  • 于是模仿它的配置，去配置我们需要的 vue.config.js
  • 这样配置完成以后，就可以在项目中使用 svg了

    去掉自带颜色

  • 有的svg图标可能有原生颜色，此时会影响我们的css,打开svg文件，找到 fill属性，删除掉该属性，即可去除
  • 批量删除fill属性，需配置使用 ‘svgo-loader’ (注：可能会失效)

抽取 Icon组件

  • 如果每次引入svg都要写一次,着实有些麻烦
  • 所以，可以抽取出一个Icon组件，设为全局属性，每次 <Icon name="money"/> 即可使用

    一次性引入所以svg 文件

  • 现在已经解决了全部的问题，如果有新的svg，就复制到assets/icons目录下，然后在项目里直接使用 即可
