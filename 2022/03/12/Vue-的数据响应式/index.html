Vue的数据响应式

|Post View:

Vue对data做了什么？

  • data 变了！ 为什么myData在传入前和传入后，两次log出的结果是不一样的呢？

    为了解决data为什么会变，需要先学习一下 getter 和 setter

  • 需求一：得到姓名

  • 需求二：函数 姓名后面的括号怎么去除

  • log出obj1后的结果如下： 会发现 姓名:(…) 和 {n(…)} 是一样的，姓名:(…)不是真的属性，而是虚拟属性，只是可以对它读写而已)

  • 于是可以猜测，{n(…)}也有一个 get 和 set 方法，

  • 上面的getter 和 setter 方法已经定义在obj1中，但是 obj1已经声明完了，如何再在obj1这个对象使用呢？

    还是不能知道 data变的原因，需要再学习一下 Object.defineProperty

  • 需求一: 用Object.defineProperty定义n

  • 需求二： n不能小于0，即 data2.n = -1无效， 但data2.n = 1有效

  • 需求三： 使用代理，解决问题

  • 需求四：解决绕过代理的问题，就算用户修改 myData,也要拦截他

    现在可以 new Vue 做了什么了

  • vm = new Vue({data:myData})

  1. 会让 vm成为 myData的代理(proxy)，也可以通过this.访问到myData
  2. 会对 myData所有属性进行监控(赋值给value,再通过getter、setter添加虚拟属性)
  3. 为什么要监控，防止myData属性变了，vm不知道
  4. vm知道了又如何？ 知道属性变了，就可以调用render(data),实现UI的刷新

小结

Object.defineProperty

  • 可以给对象添加属性、value
  • 可以给对象添加 getter / setter
  • getter / setter 用于对属性的读写进行监控

代理(proxy)

  • 对 myData对象的属性读写，全权由另一个对象 vm负责
  • 那么 vm就是 myData的代理
  • 如：不用 myData.n,偏用 vm.n(或 this.n) 来操作 myData.n

数据响应式

Vue的 data是响应式

  • const vm = new Vue({data:{n:0})
  • 如果修改 vm.n,那么 UI中的 n 就会响应我
  • Vue 2 通过 Object.defineProperty 来实现数据响应式

Vue的 data的 bug

  • Object.defineProperty 的问题
  • Object.defineProperty(obj,’n’,{…}) 必须要有一个’n’,才能监听& 代理obj.n,如果一个水前端没有给 n怎么办？
  • 仔细看上述代码，如果b不存在，Vue就不能对它监听&代理

    解决办法：

  1. 把key都声明好，后面不再加属性不就行了
  2. 使用 vue.set & this.$set

data中有数组怎么办？

  • 如果data中有数组，就没法提前声明所有key,难道只能使用 Vue.set吗？

    那 push是怎么被篡改的呢？

  • oldLength 和 this.length(newLength),监听了新增的length,然后遍历，再使用Vue.set

总结

对象中新增的 key

  1. 对于新增的 key,Vue 没有办法事先监听和代理
  2. 要使用 set 来新增 key,创建监听和代理，更新 UI
  3. 最好提前把属性都写出来，不要新增 key
  4. 但数组做不到 不新增 key

数组中新增的 key

  1. 数组中新增 key,也可以用 set 来新增
  2. 不过尤雨溪 篡改了 7个 API 方便对数组进行增删
  3. 这7个 API会 自动处理监听和代理，并更新UI
  4. 结论： 数组新增 key,最好通过 7个 API
